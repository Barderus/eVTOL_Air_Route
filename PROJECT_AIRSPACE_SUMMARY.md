# Project Airspace Summary

This file is the short handoff reference for the airport-airspace assumptions
used by the Chicago eVTOL routing project.

It documents what the code currently assumes. It does not claim to be a full
FAA operational interpretation.

## Scope

The airspace model is used to build a static airspace-related cost layer inside
the routing grid generated by `2D/make_grid.py`.

That layer is later reused by the route-generation scripts in `2D/`.

## Current Modeling Envelope

- low-altitude modeling assumption: `3000 ft AGL`
- corresponding ceiling noted in project docs: `4000 ft MSL`

These values are project assumptions for the risk surface.

## Active Grid Artifact

- current grid artifact: `geojson/risk_grid_v7.geojson`
- grid cell size: `500 m`

## Airspace Cost Components

The static airspace cost stored on each grid cell comes from two pieces:

1. Shelf or polygon airspace geometry near each modeled airport
2. Runway-aligned corridor risk near airport approach and departure axes

These are combined into:

- `airport_risk_combined`

## Constants Used In The Current Model

The code currently uses these airspace-related constants in `2D/make_grid.py`:

- `AIRSPACE_HIGH_VAL = 1.0`
- `AIRSPACE_MED_PEAK = 0.6`
- `AIRSPACE_DECAY_S_M = 8000.0`
- `AIRSPACE_DECAY_MAX_DISTANCE_M = 1500.0`

The last value is the hard cutoff for radial decay beyond the modeled airport
footprint.

## Airport-Specific Assumptions

### O'Hare

Modeled as coded shelf geometry rather than a simple circle.

Current structure:

- high-cost core uses a `5 NM` ring with `6 NM` arc geometry
- outer shelf extends to `10 NM`
- overlap transition toward the Midway area uses coded connector geometry

Risk behavior:

- core cells receive high airspace risk
- outer shelf cells receive medium airspace risk
- cells just outside the modeled shelf can receive decayed medium risk for up
  to `1500 m`

### Midway

Modeled as a core plus an outer shelf.

Current structure:

- `5 NM` core
- `10 NM` outer shelf
- overlap handling with O'Hare is encoded directly in the geometry logic

Risk behavior:

- `5 NM` core cells receive high airspace risk
- `5-10 NM` shelf cells receive medium airspace risk
- cells beyond the shelf can receive decayed medium risk for up to `1500 m`

### Lewis University Airport

Modeled as a Class D polygon rather than a circle.

Current structure:

- Class D footprint
- Clow-facing cutout encoded as a wedge subtraction

Risk behavior:

- cells inside the polygon receive high airspace risk
- cells outside the polygon can receive decayed medium risk for up to `1500 m`

## Runway Corridor Risk

In addition to shelf geometry, the project adds runway-aligned corridor risk
for airports listed in `AIRPORT_SITES`.

That corridor term depends on:

- distance along the runway axis
- distance away from the runway axis
- airport-specific peak and decay settings

This corridor risk is summed with the shelf or polygon term to form
`airport_risk_combined`.

## Static Map Classification

The static map converts `airport_risk_combined` into three display classes:

- Low: `airport_risk_combined <= 30`
- Medium: `30 < airport_risk_combined <= 70`
- High: `airport_risk_combined > 70`

## Routing Weight

When the airspace layer is used in the combined A* route cost, its current
weight is:

- Airspace = `1.4`
